
# План доопрацювання розділу «Процеси»

Обсяг великий — пропоную розбити на 4 етапи. Після кожного зупиняюся, ви тестуєте, потім переходимо далі. Нижче — що входить у кожен етап і як це виглядатиме.

---

## Етап 1. Категорії + Шаблони процесів

**UI**
- У шапці розділу `/processes` — таби: «Мої процеси», «Активні запуски», «Шаблони».
- Ліворуч на вкладці «Мої процеси» — панель категорій (згортається на мобільних). Категорії створюються адміном, кожна має назву, колір, іконку.
- Кнопка «Створити з шаблону» відкриває модалку з бібліотекою шаблонів: «Відпустка», «Оплата рахунку», «Погодження договору», «Відрядження», «Прийом на роботу» + користувацькі шаблони.

**База**
- `process_categories` (name, color, icon, sort_order)
- `processes.category_id` → FK
- `process_templates` (name, description, category_id, is_system, template_data jsonb) — зберігає всю структуру процесу (поля + кроки) для швидкого клонування

## Етап 2. Активні запуски (окрема вкладка)

**UI** — вкладка «Активні запуски»:
- Дошка/таблиця з усіма запущеними `process_runs` по всіх процесах.
- Фільтри: тип процесу, категорія, постановник, поточний виконавець, статус, дедлайн.
- Групування-перемикач: за типом / за виконавцем / за постановником / за статусом.
- Картка запуску: назва процесу, поточний крок, відповідальний (аватар), прогрес-бар по кроках, SLA-індикатор (зелений/жовтий/червоний), дедлайн.
- Швидкі дії просто з картки: «Погодити», «Відхилити», «Відкрити».

---

## Етап 3. Візуальний конструктор кроків (ядро)

Замість плоского списку полів — послідовність **кроків** із відповідальними. Простіше за n8n: лінійний потік з відгалуженнями за умовами.

**Редактор** (нова сторінка `/processes/:id/builder`):
- Ліва панель — палітра блоків, які перетягуються на канву:
  - **Ініціатор** (стартовий, автоматично)
  - **Заповнення форми** — набір полів (те, що зараз є)
  - **Погодження** — конкретна людина / роль / керівник відділу
  - **Умова (If/Else)** — розгалуження за значенням поля (напр. сума > 50 000)
  - **Автодія** — створити задачу / надіслати email / згенерувати PDF / оновити поле
  - **Завершення** (кінцевий)
- Канва — react-flow (легкий візуальний редактор, схожий на n8n, але з наперед визначеними типами вузлів). З'єднання стрілками, drag-to-connect.
- Кастомізація блоків: колір, форма (прямокутник/пігулка/ромб для умов), іконка, підпис.
- Права панель — властивості вибраного блоку (виконавець, поля, SLA у годинах, умови).

**База**
- `process_steps` (process_id, step_type, title, config jsonb, position_x, position_y, style jsonb, sla_hours)
- `process_step_transitions` (from_step_id, to_step_id, condition jsonb, label) — ребра графа

## Етап 4. Виконання, погодження, SLA, автодії

**Модель виконання**
- `process_runs.current_step_id` — де зараз запуск.
- `process_run_steps` — історія проходження: крок, виконавець, дія, коментар, час входу/виходу.
- Дії на кроці погодження: **Погодити / Відхилити / Повернути на доопрацювання** + обов'язковий коментар при відмові.
- Рушій: після завершення кроку рахує наступний вузол за transitions, обчислює умови, призначає нового відповідального, шле сповіщення.

**SLA / нагадування**
- Поле `sla_hours` на кроці. Cron-функція `check-process-sla` (раз на годину) шле нагадування виконавцю та постановнику при 50%, 100%, 150% від SLA.
- Візуальний індикатор на картці запуску.

**Умовна логіка**
- Вузол «Умова» з правилами вигляду `{ field: "amount", op: ">", value: 50000 }` (комбінувати AND/OR). Два виходи: true / false.

**Автодії** (виконуються Edge Function):
- Створити задачу (у вибраному проекті, з виконавцем)
- Надіслати email (шаблон + отримувач)
- Згенерувати PDF (за шаблоном, зберегти у вкладення запуску)
- Оновити поле процесу

---

## Технічні деталі

- Візуальний редактор: `@xyflow/react` (react-flow), кастомні `nodeTypes` під кожен тип блоку, `edgeTypes` з мітками умов.
- Рушій виконання — серверний, у Edge Function `process-engine` (advance step, evaluate conditions, run actions).
- SLA — pg_cron + Edge Function `check-process-sla`.
- Усі нові таблиці — з RLS: доступ мають постановник, поточний виконавець, учасники запуску, адміни.
- i18n — усі нові рядки в `LanguageContext.tsx` (uk/en/ru).
- Кастомізація блоків через `style jsonb` (колір фону, рамки, форма).
